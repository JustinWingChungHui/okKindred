{% trans "How it started" %}

{% blocktrans %} Hello. I'm Justin on the left. And I'm Adrian on the right. {% endblocktrans %}

{% blocktrans %} This is a website to record our family tree and act a bit like a social network. You can maintain a profile on here and discover how far the tree reaches! As a person is added to the family tree, they receive an email so they may become a user and add more people to it. The site was inspired by our family gatherings and the difficulties of explaining how we are all related. Our family is now very diverse with many ethnicities and has spread around the globe. As the family grows and evolves, we wanted a way to document everyone and keep in touch. {% endblocktrans %}

{% trans "FAQ" %}

{% trans "Can anyone see my family's information?" %}

{% blocktrans %} No, only people within your family tree that you have invited can see the information you input. The information is also not searchable by outside users or search engines. {% endblocktrans %}

{% trans "Is my information secure?" %}

{% blocktrans %} We take security and privacy very seriously. We keep the details of our family on this site, and we would never do anything to jeopardise this data! Only your own family members that you invite to share your tree are able to see the information that you enter on the site. Furthermore, we do not sell, trade, or rent your personal identification information to others. {% endblocktrans %}

{% blocktrans %} The site is hosted with PythonAnywhere who themselves take privacy very seriously. We also use Amazon S3 to store any images that you upload. We do use third party geocoding services (Google & Microsoft). This is to convert a location into a longitude and latitude so we can plot it on a map. However no other identifying pieces of information (e.g. name) are supplied with this location to these third parties. So for example we will send solely the term 'Hong Kong' to Google who will then give us a latitude and longitude so we can plot it on a map. {% endblocktrans %}

{% blocktrans %} If you are concerned, the site is entirely open source, so you can take a copy of the source code (see below), have a look how your information is stored or even host your own version of this website. We will be more than happy to help you set it up. {% endblocktrans %}

{% trans "Can I see the source code?" %}

{% blocktrans %} The source code is located at Github.com. What you see on okKindred.com is taken from the 'production' branch. {% endblocktrans %}

{% blocktrans %} We've issued it with an MIT licence which means you are free to use, copy and modify the code for your own purpose. The README contains a basic overview on how it works and step by step instructions to deploy your own instance. {% endblocktrans %}

{% blocktrans %} We welcome anyone who has any ideas or who wishes to contribute to this project and make it better! We especially need help with translation into other languages. Please get in touch if you would like to contribute, or (even better) make a pull request on Github :) {% endblocktrans %}

{% trans "What's in it for you if it's free?" %}

{% blocktrans %} We are software developers, and this is a project created in our spare time to expand our skillset and to add to our portfolio to show off to others. This was the best way to organise information on our own family, and we hope that new users will offer suggestions to improve the site or even contribute code. {% endblocktrans %}

{% trans "Can I manage multiple family trees?" %}

{% blocktrans %} No you can't! At least not yet. We are aware that this is something people may want, and may put this feature in the future. {% endblocktrans %}

{% trans "Is there an iOS/Android app?" %}

{% blocktrans %} No there is not yet. However, you will find that the website formats very nicely on a mobile device. If you are using Chrome on a mobile, you can add a shortcut of the website to your desktop which makes it look like an app for those in your family who are less tech savvy! {% endblocktrans %}

{% trans "How can I show more family members in the tree?" %}

{% blocktrans %} The standard view just shows the one level of relation. However, if you go to the eldest relative's profile, you can 'Show Descendents'. This will show all descendents with blood relations but not their partners. {% endblocktrans %}

{% blocktrans %} Displaying a family tree on a mobile device without it turning into a mess is a challenge that is yet to be resolved. {% endblocktrans %}

{% trans "What browsers does this work on?" %}

{% blocktrans %} The site is tested on the latest version of the top browsers on desktop and mobile. There may be issues on unsupported browsers, such as older Android browsers, Internet Explorer 9 and below. {% endblocktrans %}

{% trans "Is this website available in Mainland China?" %}

{% blocktrans %} As far as we know it is, but availability of certain websites in China changes day to day. We have made some technical decisions to make it more likely to be available. For instance we use Mapbox for mapping instead of Google maps because the latter is blocked in China over a secure connection. {% endblocktrans %}

{% trans "Is there a limit on images I can upload?" %}

{% blocktrans %} No at the moment there isn't. The images are stored on Amazon S3 which is very very cheap so we don't mind footing the cost. However if a lot of users start to upload thousands and thousands of images, we will have to create a limit. {% endblocktrans %}

{% trans "Can I advertise on your site?" %}

{% blocktrans %} We have no plans to raise money via ads on the site mainly as we don't want to subject our Grandmas to any more advertising than they currently get. {% endblocktrans %}

{% trans "I found a bug!" %}

{% blocktrans %} Please let us know! You can email us or you can add it to the issues list on Github. If it's a security bug need to know ASAP so we can fix it! {% endblocktrans %}

{% trans "Can you add a feature so it can do xxxxx please?" %}

{% blocktrans %} Feel free to request new features via . We will definitely consider them especially if it will benefit a lot of people (i.e. our family). This site is done in our spare time, so feature requests will be done when they're done. {% endblocktrans %}

{% trans "Other genealogy websites look up local records to help create the tree, can you do the same?" %}

{% blocktrans %} We have no plans to implement this yet. The reason being is that most of our family originates from China. Because of the civil wars that occured in China in the 20th century and instability during the last decades of the Qing dynasty, few usable records exist that would be beneficial to us. {% endblocktrans %}

{% blocktrans %} Given that many genealogy websites do this already, we would try to focus more on the social networking side of this project. However that is not to say that if in time a great need became apparent, we would not look at implementing such a feature. {% endblocktrans %}

{% trans "For describing relationships you have 'Partnered' but not 'Married', you have 'Raised' but not 'Given Birth to'?" %}

{% blocktrans %} We've tried to keep the site as simple as possible and as inclusive as possible. The definitions of 'Raised', 'Raised by' and 'Partnered' give enough information to build a tree. They are inclusive of cross-cultural relationships which don't have a concept of marriage. They can include civil partnerships, common law partners, divorcees, adoptees and any other definitions that may or may not exist in different cultures. {% endblocktrans %}
